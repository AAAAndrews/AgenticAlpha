# Anthropic Skill 编写与创建指南 (Skills Authoring Guide)

本指南基于 `anthropics/skills` 官方仓库的文档，为您提供编写和创建供 Claude 使用的 Skill（技能）的核心概念、最佳实践以及标准化步骤。

## 1. 核心概念与技能结构 (Anatomy of a Skill)

一个 Skill 本质上是一个包含核心配置和可选指令模块的文件夹结构。其核心构成如下：

*   **`SKILL.md` (必须)**：技能的核心定义文件。
    *   **YAML Frontmatter**：位于文件顶部，必须包含 `name` (技能名称) 和 `description` (技能描述)。
    *   **指令主体**：位于 Frontmatter 下方的具体 Markdown 格式的指令内容，指导 Claude 执行任务。
*   **捆绑资源 (可选)**：技能目录内可以包含其他关联资产以支持复杂的逻辑。
    *   `scripts/`：用于确定性任务或自动化处理的脚本文件。
    *   `references/`：供 Claude 参考的详尽文档或多域配置文件。
    *   `assets/`：诸如模板、图片等媒体资产。

## 2. 核心机制：触发与描述 (Triggering & Description)

技能的触发依赖于对其描述的语义匹配。

*   **描述是关键**：Frontmatter 中的 `description` 字段是 Claude 决定是否在对话中调用该技能的**唯一**依据。
*   **编写“侵入性 (Pushy)”描述**：鉴于 Claude 本身具备强大的原生推理能力，有时可能会出现对自定义技能“触发不足”的情况。最佳实践是让描述具有“侵入性”和明确性。
    *   不仅要说明技能的能力（"做什么"），还要明确指示“何时使用”。
    *   例如：“当用户提到 XYZ、数据可视化或要求相关内部指标时，务必使用此技能”。
*   **限制与格式规范**：
    *   描述内容的长度最多为 **1024 个字符**。
    *   描述内容中**严禁包含尖括号 (`<` 或 `>`)**，这可能会干扰内部解析机制。

## 3. 最佳实践与原则 (Best Practices)

### 3.1 渐进式展示 (Progressive Disclosure)

为了优化 Token 消耗和上下文窗口，Claude 处理技能时遵循三级上下文加载机制：

1.  **元数据 (Metadata)**：包括技能名称和描述。这部分数据（约100字符内）会永久保留在系统上下文中。
2.  **SKILL.md 主体**：只有当技能被成功触发时，指令主体才会被加载到当前对话的上下文中。建议将该文件理想情况下保持在 **500行以内**。
3.  **外部资源 (External Resources)**：按需提取机制。如果指令极其复杂或需要支持多领域框架（例如，同时包含 GCP 和 Azure 的部署方案），应当将详细内容分拆至 `references/` 目录中，并在 `SKILL.md` 中给出清晰的目录或检索索引，让 Claude 根据需要阅读特定文件。

### 3.2 无意外原则 (Principle of Lack of Surprise)

技能的执行过程必须符合用户的预期并保证安全性。
*   严禁隐藏意图。
*   严禁执行恶意软件、包含后门或设计越权/提权漏洞代码。
*   开发者应确保技能的行为是透明且可预测的。

## 4. 创建新技能的标准化步骤 (Creation Loop)

开发一个高质量的 Skill 是一个迭代的过程，通常分为以下 5 个步骤：

1.  **意图捕捉 (Capture Intent)**
    *   明确业务目标：这个技能用来解决什么具体问题？
    *   定义触发条件：在何种上下文或用户提问下应该触发此技能？
    *   确定预期输出格式。

2.  **编写 `SKILL.md`**
    *   定义 YAML 格式的 `name` 与 `description`。
    *   填入引导 Claude 处理目标的逐步 Markdown 指令。确保指令逻辑清晰、无歧义。

3.  **设置与运行测试 (Test Cases/Evals)**
    *   针对具有客观可验证输出的技能，设计标准化的测试用例。
    *   将包含不同场景的测试提示词输入给“已加载该技能的 Claude”。
    *   观察其触发行为及执行准确性。对于复杂项目，可考虑建立定量分析或进行 A/B 并行对比验证。

4.  **迭代优化 (Iterate & Refine)**
    *   基于测试输出或定性反馈进行调整。
    *   修改 `SKILL.md` 中表述模糊的说明。
    *   增加对边界情况 (Edge cases) 和异常处理的明确指导。
    *   使用描述优化工具微调触发的精准度。重复第 3 步和第 4 步，直至结果令人满意。

5.  **打包与部署 (Package)**
    *   使用相应的构建脚本遍历整个技能文件夹。
    *   在通过必要的内部链接和格式校验后，将其封包为 `.skill` 文件（标准的 ZIP 格式压缩包）。
    *   将打包好的文件分发给用户进行本地安装和使用。
